Diagnosis and cause
- Error type: pnpm lockfile/config mismatch during dependency installation.
- Cause: The repository's pnpm lockfile (pnpm-lock.yaml) is out of sync with the current package configuration (likely an "overrides" entry or other dependency-related change in package.json). Netlify runs a frozen install and pnpm refuses to proceed when the lockfile doesn't match package.json. The log itself recommends running pnpm install with no frozen lockfile to regenerate the lockfile.

Solution (two options — preferred is to update and commit the lockfile)

1) Preferred — update the lockfile locally and commit it (recommended for reproducible builds)
- Locally run:
```sh
corepack enable
pnpm install --no-frozen-lockfile
```
- Inspect changes, then commit and push the updated lockfile (and package.json if it changed):
```sh
git add pnpm-lock.yaml package.json
git commit -m "Update pnpm lockfile to match overrides/package.json"
git push
```
- Trigger a new Netlify build.

Also verify the lockfile is present in the repo root (pnpm-lock.yaml) and was committed before pushing.

2) Immediate Netlify build change (if you cannot update the repo lockfile right now)
- In Netlify Build settings, change the build command to run pnpm without frozen mode and remove the trailing background ampersand (&). For example:
```
corepack enable && pnpm install --no-frozen-lockfile && pnpm run build
```
Notes:
- The logs show the build command originally ends with a trailing "&" which runs install in the background; remove that so the install completes before the build step.
- After using option 2, prefer to follow option 1: update & commit the lockfile so future builds are deterministic.

Extra checks
- Confirm pnpm-lock.yaml exists in the repo root and is up-to-date.
- If you decide to pin a pnpm version for consistency, ensure your local and CI pnpm versions are compatible; otherwise regenerate and commit the lockfile using the pnpm version you intend to use.

This will resolve the ERR_PNPM_LOCKFILE_CONFIG_MISMATCH and allow Netlify to install dependencies and complete the build.
